## 基本原則

技術情報の正確性を保持しながら、読者との共感を生み出し、親しみやすさと明確な価値観を併せ持つ文体を構築する。

## 話し言葉とカジュアルな表現

### 「〜ですね」「〜でしょう」の活用

技術的な内容を語る際にも、読者に語りかける口調を維持する。

```
なんだかんだで元のマークダウンファイルっていうのは、やはり偉大だなでしょう。
ただのテキストファイルですからいろいろなことが簡単にできたりするんですよね。
```

```
しかし知識の保管庫という観点では、テキストファイルでいろいろな情報を簡単に紐付けられる体験は結構好きなので今後も使っていきたいでしょう。
```

### 内面の吐露

「思いました」「〜のです」という表現で、筆者の内面を素直に表出する。

```
そこで、「これは認知負荷が高すぎる...もう無理だ!」と正直思いました。
```

```
これは鶏が先か卵が先かのような状況です。
```

### 率直な表現

技術的な判断や現状認識を、飾らず率直に述べる。

```
正直なところを言うとまだちゃんと活用できている気がしません。
```

```
あとはシンプルに、個人のブログで SEO を実際に学んでみて検索に引っかかるようにする方が、個人の学習としては質が良いと思い、この方式を採用しています。
```

## 一人称の使用

### 「私」を主人公にした語り

技術記事であっても、筆者自身の経験を中心に据える。

```
私は過去プロジェクトのコードを流用していました。
```

```
私たち人間は、1 日の中で数多くの意思決定を行っています。
```

```
私の体験ですが、実際に契約している有料メディアの中で、音声配信(ラジオ)や文字起こしの他に、記事の内容を要約した図解を提供しているものがあります。
```

## 問いかけと共感の形式

### 読者への問いかけで導入

記事の冒頭や重要な論点において、読者に直接問いかける。

```
皆さんは、Google 製の IDE である「Antigravity」を使っていますか？
```

```
`page.evaluate()` と `page.locator().click()` の違いは何でしょうか？
```

### 共感を誘う表現

読者の経験を想起させ、共感の紐帯を構築する。

```
これを読んでいるあなたも、同じように考えているはずです。
```

```
文章で丁寧に書いたにもかかわらず、結局ほとんど読まれていなかった、という経験は心当たりがあるでしょう。
```

## 避けるべき表現

### AI生成文章の典型的な特徴

以下の表現は使用しない

- 日本語文末の「:」(コロン)
- 「詳しくはこちら👉」のような機械的な誘導
- 過度に丁寧で無機質な表現
- 「〜することができます」の連続使用
- 感嘆符(!)の過度な使用
- 倒置法(語順を入れ替えて結論や述語を先に持ってくる書き方)

### 倒置法の禁止

読みやすさの観点から、倒置法ではなく通常の語順(順序法)で書く。

```
❌ 技術的には実現可能だが、最大の壁となったのが文脈解析だった。
✅ 文脈解析が最大の壁で、技術的には実現可能だが難しかった。
```

```
❌ 面白いと思ったのが、2つのバイアスのギャップ検出という着眼点だ。
✅ 2つのバイアスのギャップ検出という着眼点は面白いと思いました。
```

### 適切な代替表現

| 避けるべき表現 | 推奨表現 |
|--------------|----------|
| 〜することができます | 〜できます / 〜します |
| 以下の通りです: | 以下の通りです。 |
| 詳しく解説します! | 詳しく解説します。 |
| ご確認ください | 確認してください / 見てください |
