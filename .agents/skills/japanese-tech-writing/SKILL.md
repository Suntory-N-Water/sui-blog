# 日本語技術文書の文章規範を適用する

## 手順

1. `references/norms.md` を読む。これが本スキルの規範本体である。
2. 対象テキスト (貼り付け本文、`contents/blog/*.md`、生成中のドラフト等) を特定する。
3. `references/norms.md` の各節 (整形 / 段落と論証の構成 / 論証の厳密さ / 読み手の負荷の管理 / 視点と語り / 演出の抑制 / LLM っぽい表現の禁止 / 冗長の排除 / 見出しの付け方 / 読者への誠実さ) の規範に照らして、違反箇所を抽出し修正案を提示する。
4. 出力は「指摘 → 該当箇所 → 修正案」の順で、規範のどの節に基づくかを明示する。

## やってはいけないこと

- 規範を独自に拡張・改変しない。`references/norms.md` の記述だけを根拠にする。
- 論旨の改変 (主張・結論・固有名詞・数値の変更) を行わない。規範違反の指摘と書き換え案にとどめる。
- 推量・可能性・読者の疑念・反実仮想として書かれている文を、機械的に断定へ変えない (`references/norms.md` の「論証の厳密さ」節に従う)。

## Error Handling

- 構成や離脱対策の推敲を求められたら、blog-revision に切り替える。
- 構成案レビューを求められたら、blog-prewriting に切り替える。
