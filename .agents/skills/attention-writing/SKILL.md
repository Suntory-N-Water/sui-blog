# アテンションライティング

## このスキルの目的

ブログ・記事・noteのように **読者が義務なく読む長文** を、最後まで読まれる文章に仕上げる。判断基準はただ一つ：

> **この行を読んだ人は、次の行を読みたくなるか。**

構成を先に固めて当てはめる従来の書き方は捨てる。冒頭から「次の1行を読みたくなるか」を問いながら書き、構成は途中で躊躇なく変える。

## いつ生成モード／推敲モードを使うか

ユーザーの依頼を見て自動で判定する：

- **生成モード**: 「〜について記事を書いて」「下書きを作って」のように、まだ本文が存在しない。
- **推敲モード**: 既存の文章(貼り付け or ファイル)があり、「直して」「書き直して」「アテンションライティング適用して」と依頼された。

どちらか曖昧なときは、ユーザーに一度だけ確認する。

## なぜ「構成ありき」を捨てるのか(書く前に頭に入れておく)

- 情報は希少ではなくなった。今希少なのは **人間の注意** である。
- 読者はお金も義務も払っていない。少しでも退屈なら一瞬で離脱する。
- 「全体構成→章立て→内容を詰める」順で書くと、書き手は伝えたいことから書き始めるが、読者は **目の前の1行** しか見ていない。ここでズレる。
- 構成は捨てない。ただし **大まかな方向だけ決めて、冒頭から書く**。次の行が読みたくなるかを問いながら書き、最初の構成は躊躇わず変える。

この「なぜ」を理解せずに下の4原則だけ機械適用すると、ただの煽り文になる。原則は手段、目的は「次の1行を読みたくさせる」こと。

## 4つの原則

### 1. いちばん美味しいものを最初に出す

出し惜しみと前置きをやめる。結論・いちばんの驚き・いちばんの面白ポイントを早く出す。ただし全部のオチを出す必要はない。「これは何の話か」「何が面白いのか」を読者が掴むまでの距離を最短にする。

- 悪い例: 「最近、文章術についてよく考えていることがあります。今日はその話をします」
- 良い例: 「構成が上手い文章ほど読まれない。これが、今の時代の真実です」

### 2. 1文を短くする

1文に1つのことだけ。長い1文は脳の負荷になり、読者は逃げる。声に出して読んだとき息継ぎが苦しいなら長すぎる。

- 「、」で繋いで複数の主張を並べない。
- 接続助詞「が」「ので」で文を伸ばさず、いったん「。」で切る。

### 3. チラ見せ(次への引っ掛かり)を残し続ける

人は「知っていること」と「知りたいこと」の隙間を埋めたくなる。書き手はその隙間を **1行ごとに作り続ける**。

- 「ここには誰も予想できない理由がありました」
- 「私たちの常識はすべて間違っていました」
- 「明日から絶対にやるべきアクションプランを3つお伝えします」

ただし乱発するとケバくなる。段落の継ぎ目や、読者が離れそうな箇所に置く。

### 4. 無駄な装飾を削る

「今からこのような話について説明していこうと思うのですけれども」のような前置き・自己言及・くどい修飾はカット。

- 「〜していこうと思います」→ 不要
- 「ちなみに」「なお」「さて」→ 多くの場合カット可
- 二重否定・婉曲表現 → 言い切る

## 書きながら(推敲しながら)問い続ける唯一の質問

> **この行を読んだ人は、次の行を読みたくなるか。**

YESなら進む。NOなら、その行を直すか、消す。構成のために残してはいけない。

## 生成モードの進め方

1. ユーザーから題材・要点・読者像を聞く(既に提示済みなら確認)。聞きすぎない。
2. 大まかな方向だけ決める(章立てを完成させない)。
3. **冒頭3行** を書く。ここがいちばん重要。「いちばん美味しいもの」を置けているか自問する。
4. 1段落書くごとに「次の段落を読みたくなる引っ掛かりが末尾にあるか」を確認する。
5. 書き終わったら、推敲モードのチェックを自分で1周かける。

## 推敲モードの進め方

既存の文章を受け取ったら、次の順で処理する：

1. **冒頭診断**: 最初の3行で「これは何の面白い話か」が伝わるか。伝わらないなら、本文中の「いちばん美味しい一文」を探して冒頭に持ち上げる。
2. **1文長チェック**: 60字を超える文を抽出し、分割案を出す(目安。短ければ短いほどよいわけではない、リズムは保つ)。
3. **離脱ポイント検出**: 段落の継ぎ目で「次を読みたくなる引っ掛かり」が無い箇所を指摘。チラ見せ文の追加案を出す。
4. **装飾削除**: 前置き・自己言及・冗長表現を列挙し、削除/言い換え案を出す。
5. **書き直し提示**: 上記を反映した全文を提示する。元の論旨・固有名詞・主張は変えない。

推敲の出力フォーマット(既存記事を直す時)：

```
## 診断
- 冒頭: (所見)
- 1文長: (長すぎる文の指摘)
- 離脱ポイント: (箇所)
- 無駄な装飾: (箇所)

## 書き直し
(全文)
```

ユーザーから「全文だけ出して」「差分だけ出して」など指示があればそれに従う。

## やってはいけないこと

- **煽り化・釣りタイトル化**: 「衝撃!」「ヤバすぎ!」のような中身のない強調はチラ見せではない。事実に基づく具体的な引っ掛かりを置く。
- **論旨の改変**: 推敲モードで主張・結論・固有名詞・数値を勝手に変えない。読みやすさのための削除と並べ替えに留める。
- **原則の機械適用**: 全文を短文で刻むと逆に読みにくい。リズムを保つために中くらいの長さの文も混ぜる。判断基準はあくまで「次の1行を読みたくなるか」。
- **章立てを先に完成させてから埋める**: これが本スキルの捨てる書き方そのもの。生成モードでは冒頭から書く。
